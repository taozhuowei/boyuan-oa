# core.bus — 模块间通信总线

## 职责

定义模块间通信规范，防止业务模块之间直接 import 产生强耦合；规范已在设计文档中确立，代码实现随各业务模块集成逐步落地。

## 主要功能（设计意图）

- 事件注册：所有跨模块异步事件继承 core.bus.events 基类，通过 Spring ApplicationEvent 机制发布与监听，触发方不直接引用接收方
- 同步查询端口：跨模块同步查询通过 core.bus.ports 中定义的接口（Port）进行，实现方在各模块内，调用方依赖接口而非实现类
- 模块边界隔离：包结构约束通过 ArchUnit 测试规则在 CI 阶段静态验证，违规直接导致构建失败

## 对外暴露的接口

无（内部机制，非 HTTP 接口）

## 依赖

无（横切规范层）

## 技术债 / 待完善

- 规范已定义，当前各模块仍存在通过 @Autowired 直接引用跨模块 Service 的情况，ArchUnit 规则尚未完全落地
- Spring ApplicationEvent 为同进程内事件总线，若未来采用微服务架构需替换为消息队列（RabbitMQ/Kafka）
