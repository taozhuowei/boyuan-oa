# 博渊 OA — 跨平台设计规范

适用范围：Web 端（H5）与微信小程序端（MP）。两端共用本文件中定义的配色、字体、间距、圆角和组件设计原则；平台差异在各自子目录的 README.md 中补充说明。

---

## 设计哲学

基于 Apple Human Interface Guidelines，结合中文企业 Web 场景适配。

**简洁克制，以任务和内容为中心。** 界面服务于用户的工作目标，装饰元素只在明确增益体验时才引入。

**信息层级清晰，减少视觉噪音。** 通过字号、字重、颜色对比建立层级，不依赖分割线和边框来填充空间。

**操作反馈即时，用户始终知道系统状态。** 提交、保存、审批等每个操作必须在 100ms 内给出加载态，在操作完成后给出明确的成功或失败提示。

**表单字段精简，只收集必要信息。** 每个字段的存在都必须有业务理由；可选字段明确标注，不得与必填字段视觉等权。

## 配色方案

浅色模式。颜色值均为 sRGB 十六进制，不支持动态颜色/暗色模式（当前阶段）。

### 主色

- 交互蓝 `#007AFF` — 所有交互元素、主按钮、链接、选中态、焦点环

### 语义色

- 成功绿 `#34C759` — 审批通过、操作成功、在线状态
- 警告橙 `#FF9500` — 待处理、临近截止、需要注意
- 错误红 `#FF3B30` — 表单校验失败、删除操作、系统错误
- 提醒黄 `#FFCC00` — 非紧急提醒、信息性标注

### 背景色

- 一级背景 `#FFFFFF` — 主内容区、卡片
- 二级背景 `#F2F2F7` — 侧边栏、输入框背景、列表底色
- 三级背景 `#EFEFF4` — 嵌套分组、禁用区域

### 文字色

- 主文字 `#000000` — 正文、标题
- 次文字 `#636366` — 辅助说明、时间戳、标签描述
- 占位文字 `#ADADB8` — 输入框 placeholder
- 分割线 `#C6C6C8` — 列表分割线、边框

## 字体

字体栈（按优先级）：

```
PingFang SC, Microsoft YaHei, -apple-system, system-ui, sans-serif
```

字阶定义：

- 正文：14px / 行高 22px / 字重 400
- 模块标题：17px / 行高 24px / 字重 600
- 页面大标题：20px / 行高 28px / 字重 700
- 辅助文字：12px / 行高 18px / 字重 400
- 表格表头：13px / 字重 500

---

## 间距

基础单位 4px，所有间距取 4 的整数倍。

- `4px` — 图标与文字间距、紧凑徽章内边距
- `8px` — 紧凑布局、标签内边距、图标按钮内边距
- `12px` — 表单项间距、卡片内边距（紧凑模式）
- `16px` — 卡片标准内边距、列表项行高补充
- `24px` — 区块间距、表单分组间距
- `32px` — 页面左右 padding

---

## 圆角

- `6px` — 小组件：标签、徽章、输入框、按钮
- `8px` — 卡片、下拉菜单、工具提示、选择器弹层
- `12px` — 大弹窗、抽屉、底部弹出面板

---

## Web 端布局结构

固定两栏布局，顶栏始终固定在视口顶部。

```
┌─────────────────────────────────────────┐
│              TopBar（56px，固定）          │
├───────────────┬─────────────────────────┤
│               │                         │
│  左侧导航栏   │      内容区             │
│  （220px）    │                         │
│               │                         │
└───────────────┴─────────────────────────┘
```

布局规则：

- TopBar 固定在顶部，高度 56px，z-index 高于所有内容层
- workbench 首页不显示左侧导航栏，内容区全宽展示模块宫格
- 进入任意模块后：左侧 220px 导航栏 + 右侧内容区
- 内容区最大宽度 1200px，左右 padding 32px

## 反馈模式

### Toast — 顶部无打扰提示

适用场景：保存成功、操作完成、复制成功等非阻塞提示。

- 位置：页面顶部居中，不遮挡主要操作区
- 持续时间：3 秒自动消失
- 不堆叠：同时只显示最新一条，旧条立即替换

### Dialog — 居中打断弹窗

适用场景：删除确认、提交审批、不可逆操作。

- 必须包含明确的"确认"和"取消"两个按钮
- 确认按钮颜色：危险操作使用错误红 `#FF3B30`，普通操作使用交互蓝 `#007AFF`
- 弹窗标题说明操作对象，正文说明后果

## 自定义组件清单

以下组件在 Web 端和小程序端均存在，实现分平台，设计语义一致。

**TopBar** — 顶栏

通知铃（带未读数徽章）、待办数徽章、个人中心（头像 + 下拉菜单）。

**ModuleTile** — 宫格模块入口卡片

包含模块图标（32px）、模块名（12px）、可选角标。workbench 首页使用。

**ApprovalTimeline** — 审批流时间线

展示审批节点状态、审批人姓名、审批时间、审批意见。节点状态颜色对应语义色。

**SignatureCanvas** — 手写签名画布

支持触摸屏和鼠标绘制，可清空重签，输出为图片 base64。

**FileUpload** — 文件上传

支持拖拽和点击两种触发方式，显示上传进度，图片文件显示缩略图预览。

## 对比度要求

遵循 WCAG 2.1 AA 级标准：

- 正文（14px，字重 400）：前景色与背景色对比度 ≥ 4.5:1
- 大字（17px，字重 600 及以上）：对比度 ≥ 3:1
- 交互蓝 `#007AFF` 在白色背景 `#FFFFFF` 上的实测对比度约 4.5:1，满足 AA
