# 博渊 OA — 全自动化浏览器测试用例设计

> **文档职责**：定义博渊 OA 平台的全自动化浏览器端测试用例，覆盖所有角色的所有业务流程。
> 供 AutoTest 工具（`tools/autotest/`）直接加载执行。
>
> **与现有测试的关系**：本文档专门描述 E2E 浏览器自动化测试用例。
> 单元测试、集成测试用例见 `../TEST_DESIGN.md`。
>
> **执行方式**：用例文件位于 `test/autotest/cases/`，由 AutoTest 工具读取并驱动。
> AI 视觉识别模式（Claude Vision）将视觉描述翻译为屏幕坐标，模拟真实人类操作。
>

## 1. Auth — 登录与认证

### 正向操作

| ID | 标题 | 角色 | 优先级 |
|---|---|---|---|
| TC-AUTH-01 | CEO 正确账号密码登录 | ceo | P0 |
| TC-AUTH-02 | HR 正确账号密码登录 | hr | P0 |
| TC-AUTH-03 | 财务正确账号密码登录 | finance | P0 |
| TC-AUTH-04 | 项目经理正确账号密码登录 | pm | P0 |
| TC-AUTH-05 | 员工正确账号密码登录 | employee | P0 |
| TC-AUTH-06 | 登录后刷新页面保持登录态 | ceo | P1 |
| TC-AUTH-07 | 点击退出登录跳转到登录页 | ceo | P1 |
| TC-AUTH-08 | 退出后 Token 失效，手动访问 /dashboard 跳回登录页 | ceo | P1 |

**TC-AUTH-01 步骤设计**
```
前置：后端运行，数据库含种子数据
1. 导航到 /login
2. 输入用户名 "ceo.demo" → 标有"用户名"或"账号"的文本输入框
3. 输入密码 "123456" → 标有"密码"的密码输入框
4. 点击蓝色"登录"主按钮
5. 断言：URL 包含 /dashboard
6. 断言 (visual)：右上角或顶部导航出现用户名称或头像
期望：跳转工作台，无报错提示
```

### 异常操作

| ID | 标题 | 输入 | 期望结果 | 优先级 |
|---|---|---|---|---|
| TC-AUTH-09 | 错误密码 | 账号正确，密码错误 | 页面显示密码错误提示，停留在登录页 | P0 |
| TC-AUTH-10 | 不存在的账号 | 账号不存在 | 页面显示账号不存在或凭证错误提示 | P0 |
| TC-AUTH-11 | 账号为空 | 账号空，密码有值 | 表单校验报错，不发起请求 | P1 |
| TC-AUTH-12 | 密码为空 | 账号有值，密码空 | 表单校验报错，不发起请求 | P1 |
| TC-AUTH-13 | 账号密码均为空 | 两者均空 | 双重报错提示 | P1 |
| TC-AUTH-14 | SQL 注入账号 | `' OR 1=1 --` | 登录失败，无服务端 500 | P1 |
| TC-AUTH-15 | 超长账号（500字符） | 超长字符串 | 不崩溃，给出错误提示 | P2 |
| TC-AUTH-16 | 特殊字符密码 | `!@#$%^&*()` | 若账号匹配则正常处理，不因特殊字符崩溃 | P2 |
| TC-AUTH-17 | Token 过期后 API 返回 401 | 手动修改 localStorage token 为过期值 | 前端自动跳转登录页 | P1 |
| TC-AUTH-18 | 已停用账号尝试登录 | 停用账号 | 提示账号已停用，无法登录 | P1 |

**TC-AUTH-09 步骤设计**
```
前置：无
1. 导航到 /login
2. 输入用户名 "ceo.demo"
3. 输入密码 "wrongpassword"
4. 点击蓝色"登录"按钮
5. 断言 (visual)：页面中出现红色错误提示文字，包含"密码"或"账号"字样
6. 断言 url_equals：仍在 /login
期望：停留登录页，显示错误提示
```

| ID | 标题 | 优先级 |
