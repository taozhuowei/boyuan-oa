# Impeccable Design Context — 众维 OA 工作台

## Design Context

### Users
建筑与工程企业内部员工，覆盖 5 类角色：普通员工、现场劳工（施工人员）、财务、项目经理、CEO。使用场景分两端：
- **Web 端（PC/平板）**：财务/CEO/项目经理 —— 配置管理、批量处理、数据查看，坐在桌前操作，注重信息密度
- **微信小程序**：全员 —— 待办审批、提交单据、查看工资条，在工地/移动场景使用，注重操作效率和触控体验

### Brand Personality
**专业 · 稳重 · 高效**

这是施工企业的内部工具，不需要讨好，需要信任感。用户每天打开它处理工资、审批、考勤，界面必须让他们感到"可靠"和"不会出错"。情感目标：**稳定感 + 专注感**，不追求视觉刺激。

### Aesthetic Direction
- **视觉基调**：深海军蓝（`#003466`）作为锚点色，白色卡片承载内容，克制的阴影
- **参考方向**：Notion 的信息密度感 + 钉钉 PC 端的专业感 + Linear 的卡片清晰度
- **反参考**：避免消费级 App 风格（过多渐变、彩色气泡、圆润卡通感）；避免偏浅的灰白风格（需要权威感）
- **主题**：亮色优先，CSS 变量结构预留 dark mode 切换能力，当前不实现
- **字重**：标题偏粗（600–700），正文保持 400–500，用字重建立层级而非字号

### Design Principles

1. **信息密度优先**：Web 端每屏承载足够信息，减少翻页。卡片内容紧凑但不拥挤，行高 1.5–1.6。

2. **操作路径清晰**：主操作按钮唯一且突出（深蓝实色），次操作用描边，破坏性操作用红色。层级不超过 3 级。

3. **双端一致但不一样**：品牌色、字体、圆角、间距 token 双端共用；但 Web 端用 Row/Col 网格，小程序端用纵向卡片列表。

4. **可信赖的细节**：WCAG AA 对比度（文字 ≥4.5:1，交互元素 ≥3:1）；状态反馈及时（loading/success/error）；边界状态有 empty state。

5. **无装饰噪音**：图标辅助文字，不替代文字；颜色用于传达状态（红=错误/高优先，黄=警告，绿=成功），不用于装饰。

```
Primary:       #003466   (深海军蓝)
Primary-dark:  #002244
Primary-light: #e6f0f7   (背景浅蓝)
Surface:       #ffffff   (卡片)
Surface-low:   #f5f5f5   (页面底色)
On-surface:    rgba(0,0,0,0.85)
Muted:         rgba(0,0,0,0.45)
Error:         #f5222d
Warning:       #faad14
Success:       #52c41a

Radius:  sm=4px  md=8px  lg=12px  xl=16px
Shadow:  sm=0 2px 8px rgba(0,0,0,0.08)  md=0 4px 16px rgba(0,0,0,0.12)
Spacing: xs=4  sm=8  md=16  lg=24  xl=32 (px)
Font:    Display=Manrope  Body=Inter  CJK fallback=PingFang SC, Microsoft YaHei
```

## Scope

- Web 端 H5 使用 Ant Design Vue 4.x，通过覆盖 `--ant-color-primary` 等 CSS 变量统一品牌色
- 小程序端使用 Vant 4.x，通过覆盖 `--van-primary-color` 等变量统一
- 自定义组件遵循上述 token，不硬编码颜色值
