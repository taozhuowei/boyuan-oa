## 1. 人员与组织架构

### 功能概述

系统以"员工"为最小单位，每位员工有且仅有一个**岗位**（决定薪资和功能权限）、一个**角色**（决定操作权限）、一个**直系领导**（决定审批路径）。

### 员工类型

| 类型 | 说明 |
|------|------|
| **OFFICE（办公室员工）** | 坐班人员，有固定岗位，可配置绩效奖金 |
| **LABOR（劳工）** | 现场施工人员，按工种管理，可提交施工日志 |

员工类型由岗位自动决定，**不需要手动设置**。

### 组织架构

每位员工有一个**直系领导**，构成公司的汇报层级树，用于确定审批人。

**直系领导确定规则（按优先级）：**
1. CEO 为员工手动指定的直系领导
2. 员工所属岗位配置的"默认上级岗位"
3. 员工所在项目的项目经理
4. CEO（最终兜底）

> **注意**：CEO 是所有员工的最终上级，任何员工没有领导时均归 CEO 管理。

### 支持
- 为每位员工单独指定直系领导（可覆盖岗位默认）
- 在组织架构图中可视化查看全员汇报关系
- 通过员工档案页面修改直系领导

### 不支持
- 一位员工拥有多个直系领导（当前版本为单线汇报）
- 系统自动根据项目变更动态更新直系领导（需手动维护）

## 2. 岗位与等级体系

### 功能概述

**岗位**是连接员工与薪资规则的核心实体。每个岗位携带薪资、假期、社保方式等配置，员工默认继承其岗位的所有规则。

**等级**是岗位的可选子配置，同一岗位可设置多个等级（如初级/中级/高级），不同等级对应不同的薪资和假期标准。

### 配置继承关系

```
全局默认 → 员工类型默认 → 岗位配置 → 等级配置 → 个人覆盖
（优先级由低到高，右侧覆盖左侧）
```

### 岗位可配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 基本工资基准 | 员工当月基础薪资 | 需手动设置 |
| 加班费计算基准 | 基本工资 / 全部工资 / 自定义金额 | 基本工资 |
| 平日加班倍数 | 工作日加班补贴倍率 | 1.5 倍 |
| 周末加班倍数 | 周末加班补贴倍率 | 2.0 倍 |
| 法定节假日倍数 | 节假日加班补贴倍率 | 3.0 倍 |
| 绩效奖金默认值 | 当月绩效奖金固定金额 | 0（开启绩效后必填）|
| 年假天数 | 年度带薪假期天数 | 需手动设置 |
| 请假扣款基准 | 请假扣款按哪部分工资计算 | 基本工资 |
| 各类请假扣款比例 | 年假/事假/病假等扣款百分比 | 年假0%、事假100%、病假50% |
| 社保处理方式 | 公司代缴 / 并入工资 | 需初始化时选择 |

### 等级配置项

等级为**可选配置**。若岗位不配置等级，则所有员工使用岗位统一值。

等级可覆盖：基本工资、绩效奖金默认值、年假天数。

### 个人覆盖

财务可为单个员工覆盖：基本工资、绩效奖金；该操作需 **CEO 审批**后生效。

### 支持
- 岗位增删改（删除前须解除所有员工绑定）
- 每个岗位配置多个等级
- 员工在岗位内跨等级调整（即时生效，不影响历史工资单）
- 社保方式切换（需 CEO 审批，**下一结算周期**生效）

### 不支持
- 一名员工同时归属多个岗位
- 岗位删除后自动迁移员工到其他岗位（需手动处理）

## 3. 考勤管理

### 3.1 请假申请

员工在系统中填写请假类型、时间段和原因，可附上证明材料。

**审批路径：** 员工提交 → 直系领导审批 → 归档

> 若直系领导当前为 CEO，则直接由 CEO 完成审批（单节点）。

**请假类型：** 年假、事假、病假、婚假、产假、其他（均可在系统中配置扣款比例）

**追溯请假：** 支持在**任何时间**补录过去的请假记录，审批通过后扣款计入**审批完成当月**的工资，不追溯历史工资单。

**配置项：**

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 最小计量单位 | 按小时 / 半天 / 天计算 | 天 |

---

### 3.2 加班管理（通知制）

**加班采用"通知制"**，由项目经理或 CEO 主动发起通知，员工不能自行申报加班。

**三种路径：**

| 情形 | 流程 |
|------|------|
| **项目经理发起通知** | PM 填写通知内容 → 发送给项目成员 → 员工默认同意 → 完成归档 → 计入当月工资 |
| **CEO 发起通知** | CEO 发送通知 → 无需审批，直接归档 |
| **员工自补申请（例外）** | 员工填写加班情况并上传证明 → 直系领导审批 → CEO 终审 |

> **员工如有异议**：员工收到加班通知后，可点击"拒绝"并说明原因，由项目经理审批（通过则确认员工未加班；驳回则视为已加班）。

**配置项：**

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 加班补贴倍率 | 在岗位中分类型配置 | 平日1.5×、周末2×、节假日3× |

### 支持
- 一次通知同时覆盖多人
- 员工拒绝加班通知（需经理确认）
- 员工自补加班（例外路径，需双审）

### 不支持
- 员工自行无审批申报加班
- 加班通知自动绑定到施工日志

## 4. 工伤补偿

### 申请与审批

任何人均可代为提交工伤申请，填写受伤员工工号、受伤情况并上传医疗单据。

**审批路径（依据提交人身份决定）：**

| 提交人 | 审批路径 |
|--------|----------|
| 劳工本人 | 项目经理初审 → CEO 终审 |
| 项目经理代提交 | 跳过初审，直接 CEO 终审 |
| 财务代提交 | 项目经理初审 → CEO 终审 |
| 其他员工代提交 | 项目经理初审 → CEO 终审 |

> **无项目经理时**：初审自动转给 CEO，CEO 可连续审批两个节点。

### 理赔金额录入

工伤审批完成后，保险理赔需要一段时间。理赔完成后由**财务**录入最终金额，并指定**计入哪个月的工资单**，与工伤发生月份可以不同，系统不强制同月结算。

### 支持
- 任何人代录工伤申请
- 理赔金额跨月入账

### 不支持
- 系统自动与保险机构对接查询理赔进度（需财务手动录入）

## 5. 施工日志

> **施工日志是独立系统**，与薪资无关，用于工程进度记录、复盘和汇报。

### 5.1 日志填报

劳工每个申报周期提交一次施工日志（默认每天）。

**填写方式：** 在表单中添加多行工作项（工作内容 + 数量 + 单位）。工作项可自由填写，也可从模板中选择预置项目快速填入。

**配置项：**

| 配置项 | 管理权限 | 默认值 | 修改是否需审批 |
|--------|----------|--------|--------------|
| 申报周期 | 项目经理修改须 CEO 审批；CEO 可直接修改 | 每天提交一次 | 是（非CEO时）|

---

### 5.2 工作项模板

项目经理和 CEO 可管理**工作项模板**，模板包含若干预设工作项（名称 + 默认单位）。

- 支持新增、编辑、删除模板
- 支持基于已有模板派生新模板（复制后修改）
- 劳工填报时可选择模板快速填入，仍可自定义修改

---

### 5.3 项目进度里程碑

项目经理可在项目中创建**里程碑节点**（如"地基完工"、"主体封顶"），在里程碑完成时标记完成日期。

每日由 PM 确认当日进度状态（正常推进 / 有风险 / 延误），CEO 可实时查看各项目里程碑进度。

---

### 5.4 单条日志审批流

劳工提交 → 项目经理审批（可通过 / 驳回 / 添加批注）→ 驳回后劳工可修改重提

> **CEO 追溯驳回**：CEO 可在**任何时间**对已归档的日志发起驳回（状态变为"已追溯驳回"），通知劳工重新提交，原日志记录完整保留。

---

### 5.5 汇总报告流程

当项目经理完成当前申报周期所有日志的审批后，可触发生成汇总报告：

1. 项目经理选择可视化展示组件（工作量折线图 / 里程碑时间轴 / 工作项汇总表格）
2. 填写 PM 总结内容
3. 提交后系统通知 CEO 查看
4. 报告自动归档，CEO 和 PM 均可随时查看历史报告

### 支持
- 按时间范围筛选汇总，系统生成对应可视化
- 钻取到某日施工日志明细（从趋势图点击）
- 历史报告在保留期内可查（默认 1 年，可配置延期）

### 不支持
- 施工日志与薪资挂钩
- 系统自动计算工程量（仅记录和展示，不做工程量计算）

## 6. 薪资结算

### 6.1 结算周期配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 结算周期 | 月结 / 半月结 | 月结 |
| 发薪日 | 每月几号发放工资 | 当月 25 日 |
| 窗口期时长 | 数据锁定前的确认窗口天数 | 7 天 |
| 窗口期提醒 | 结算截止日前多少天提醒财务 | 7 天 |

---

### 6.2 窗口期模型

传统预结算"全员阻塞"的模式已被**窗口期模型**取代。

**运作方式：**
1. 结算周期结束后，系统开启一个窗口期（默认 7 天）
2. 窗口期内：员工可补录考勤异常，PM 可补发加班通知，员工可自补加班（例外路径）
3. 窗口期结束后，**数据锁定**：
   - 未处理的考勤异常 → 按实际记录处理（未录入则按规则扣款）
   - 未审批完成的加班通知 → 不计入本期加班补贴
   - 未结算的工伤补偿 → 顺延至下一期结算
4. 财务在数据锁定后即可执行正式结算

> **窗口期可由 CEO 提前关闭**（例如快速结算时）。

---

### 6.3 结算步骤

1. **财务执行结算**：系统检查以下项目
   - 窗口期已关闭
   - 所有员工薪资档案已配置

2. **生成工资单**：系统自动为每位员工计算并生成工资单

3. **员工签名确认**：工资单发布后，员工可查看并电子签名确认（需提前完成签名绑定和 PIN 码设置）

---

### 6.4 工资组成

| 项目 | 来源 |
|------|------|
| 基本工资 | 岗位 / 等级配置，个人可覆盖（需 CEO 审批）|
| 加班补贴 | 系统自动计算（按通知归档记录）|
| 工伤补偿 | 财务手动录入，可跨月指定 |
| 绩效奖金 | 岗位 / 等级配置，财务可覆盖（需 CEO 审批）|
| 请假扣款 | 系统自动计算（按岗位配置比例）|
| 社保（个人部分）| 系统自动计算（公司代缴模式）或固定补贴（并入工资模式）|
| 个人所得税 | 系统自动计算 |
| **实发工资** | 以上各项汇总 |

**社保两种模式说明：**

| 模式 | 工资条显示 | 员工责任 |
|------|------------|----------|
| **公司代缴** | 显示各险种个人部分扣除明细 | 无需自行办理 |
| **并入工资** | 只显示"五险一金补贴（合计）"一行 | 须自行前往社保机构办理**灵活就业**参保手续 |

---

### 6.5 工资确认协议

员工在电子签名确认工资条时，需先阅读并同意**工资确认协议**。

- 协议由 CEO 在系统初始化时上传（可选），也可随时修改
- 若未上传协议，员工签名时跳过阅读步骤，直接确认
- 协议修改后，下一次签名时员工须重新同意

---

### 6.6 结算更正

正式结算后如需修改，流程如下：

财务填写更正原因 → CEO 审批 → 审批通过后周期解锁 → 财务重新结算 → 生成新版工资单 → 员工重新签名确认

历史版本工资单完整保留，不删除。

---

### 6.7 窗口期关闭后的数据锁定（无例外申请）

窗口期到期后自动锁定，**不提供例外申请机制**。未完成数据按锁定规则自动处理（见 WORKFLOW_CONFIG §2.2.2）：

- 考勤异常未处理 → 按缺勤扣款
- 加班/请假审批未完成 → 计入审批完成当月，不阻塞本月结算
- **不可豁免的强制校验**（2 项，见 WORKFLOW_CONFIG §2.2.3）：无未处理异议单、无进行中计算任务
- **不可例外**：员工薪资档案未配置（必须补齐才能结算）

## 7. 角色与权限

### 内置角色说明

| 角色 | 主要职责 |
|------|----------|
| **CEO** | 全局数据访问；所有业务终审；系统配置（审批流、数据保留策略）；提前关闭窗口期；施工日志追溯驳回 |
| **财务** | 薪资结算；工伤理赔录入；员工档案协管；发起结算更正申请；发起窗口期例外申请 |
| **项目经理** | 项目范围内初审（请假/工伤）；施工日志审批与模板管理；进度里程碑管理；汇总报告生成；发起加班通知 |
| **员工** | 发起本人请假申请；确认/拒绝加班通知；查看本人工资条；代录工伤申请 |
| **劳工** | 提交施工日志；发起工伤申请；确认/拒绝加班通知；查看本人工资条 |

### 自定义角色

CEO 可在内置角色基础上新增自定义角色，从权限集中勾选所需权限。

**限制：** 终审审批、解锁结算更正、配置审批流、配置数据保留策略这四项高风险权限**不可分配给自定义角色**。

### 支持
- 自定义角色增删改
- 每人一个角色（当前版本）

### 不支持
- 一名员工同时拥有多个角色

## 8. 数据管理

### 数据保留策略

所有数据默认保留 **1 年**，敏感数据（工伤记录、工资确认存证）默认保留 **10 年**。CEO 可按数据类型单独调整保留期限。

**到期前 30 天**，系统自动提醒 CEO：涉及数据类型、预计到期日期、数量。

CEO 可选择：
- **导出后删除**：系统自动打包导出，完成后执行清理
- **延期**：延长保留期限
- **忽略**：到期后系统自动清理，不再提醒

**注意**：不存在"永久保留"类型，所有数据均有期限，以满足数据合规要求。
