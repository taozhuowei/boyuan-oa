# 博渊·建筑工程版 — 工作流与业务规则配置

> **文档职责**：定义建筑工程版预置方案的审批流配置、薪资计算规则和数据保留策略。这些是平台工作流引擎的配置输入，而非平台本身的实现。
>
> **目标读者**：产品经理、实施顾问、后端开发者（种子数据参考）。
>
> **不包含内容**：工作流引擎的实现原理（见 `platform/ARCHITECTURE.md §5.2`）；角色定义（见 `ROLE_CONFIG.md`）。

## 1. 审批流配置

### 1.1 请假申请（LEAVE）

```
提交 → 节点1: 初审（project_manager，项目范围内）
     → 节点2: 终审（ceo）
     → 通过 / 驳回
```

| 字段 | 值 |
|------|-----|
| 业务类型 | `LEAVE` |
| 节点数量 | 2 |
| 节点1 | 审批人类型: `ROLE`，角色: `project_manager`，数据范围: `PROJECT` |
| 节点2 | 审批人类型: `ROLE`，角色: `ceo`，数据范围: `GLOBAL` |
| 驳回策略 | 保留历史，申请人可重新发起 |
| 施工日志例外 | 仅节点1（初审），无终审 |

### 1.2 加班申请（OVERTIME）

与请假申请相同的两节点流程。

| 字段 | 值 |
|------|-----|
| 业务类型 | `OVERTIME` |
| 节点1 | `project_manager`，项目范围内 |
| 节点2 | `ceo`，全局 |

### 1.3 工伤补偿（INJURY）

```
提交（worker 本人 / finance / project_manager 代录）
  → 节点1: 初审（project_manager 或 finance，看录入人）
  → 节点2: 终审（ceo）
  → 通过后金额自动纳入当月薪资结算
```

| 字段 | 值 |
|------|-----|
| 业务类型 | `INJURY` |
| 节点1 | `project_manager`（同项目），或 `finance`（跨项目代录时） |
| 节点2 | `ceo`，全局 |
| 薪资联动 | 通过后 `compensationAmount` 自动写入工资单 `injury` 项 |

### 1.4 施工日志（CONSTRUCTION_LOG）

```
劳工提交 → 节点1: 审批（project_manager，同项目）→ 通过 / 驳回
```

| 字段 | 值 |
|------|-----|
| 业务类型 | `CONSTRUCTION_LOG` |
| 节点数量 | 1（无终审） |
| 节点1 | `project_manager`，项目范围内 |

### 1.5 工资条异议（PAYROLL_DISPUTE）

```
员工发起异议 → 财务核对 → 财务发起更正申请
  → CEO 审批（解锁周期）→ 财务重新结算 → 员工重新确认
```

| 字段 | 值 |
|------|-----|
| 业务类型 | `PAYROLL_DISPUTE` |
| 特殊说明 | 此流程触发的是周期解锁，而非标准单据审批流 |

### 1.6 配置约束

- 当前版本不支持条件分支（如金额超过阈值走不同节点）
- 不支持并行会签
- 审批流变更仅对变更后新提交的单据生效，历史在途单据保持原流程

## 2. 薪资规则配置

### 2.1 结算周期

| 字段 | 值 |
|------|-----|
| 周期类型 | 月结（`MONTHLY`） |
| 发薪日 | 每月 10 日（可配置） |
| 结算触发 | 财务手动发起，非自动触发 |

### 2.2 工资项构成

| 工资项 | 类型 | 适用对象 | 计算方式 |
|--------|------|---------|---------|
| 基本工资 | 固定 | 全员 | 员工档案中配置的月固定工资 |
| 绩效奖金 | 浮动 | OFFICE | 财务按周期手动录入 |
| 加班补贴 | 浮动 | 全员 | 平日：基本工资 / 21.75 / 8 × 1.5 × 加班时长；周末：× 2；节假日：× 3 |
| 请假扣款 | 扣款 | 全员 | 基本工资 / 21.75 × 请假天数（病假减半） |
| 工伤补偿 | 补贴 | LABOR | 已通过的工伤补偿申请金额（直接录入） |
| 社保（个人） | 扣款 | 全员 | 按当地标准配置，默认 8%（养老 8%、医疗 2%、失业 0.5%） |
| 住房公积金（个人） | 扣款 | 全员 | 默认 5%，可按档案配置 |
| 个人所得税 | 扣款 | 全员 | 按累计预扣法计算，平台引擎执行 |
| 实发工资 | 汇总 | 全员 | 基本 + 绩效 + 加班 + 工伤 - 请假 - 社保 - 公积金 - 个税 |

### 2.3 预结算校验项（5 项，全部通过才能正式结算）

1. **考勤完整性**：周期内所有员工的考勤数据已录入，无未处理异常
2. **审批完整性**：周期内所有请假 / 加班单据已审批完结（无 PENDING/APPROVING 状态）
3. **日志完整性**：周期内所有劳工的施工日志已审批完结
4. **工伤完整性**：周期内所有工伤补偿申请已处理完结
5. **档案完整性**：所有员工的薪资档案（基本工资、社保比例等）均已配置

### 2.4 更正申请流程

```
财务发起更正申请（填写更正原因）
  → CEO 审批通过 → 周期解锁
  → 财务重新结算（生成新版本，版本号 +1）
  → 员工重新确认
  → 历史版本完整保留，不可篡改
```

## 3. 数据保留策略配置

### 3.1 默认保留期（Sysadmin 初始化时预置，CEO 可修改）

| 数据类型 | 保留期 | 特殊说明 |
|---------|--------|---------|
| 请假记录 | 5 年 | |
| 加班记录 | 5 年 | |
| 工伤补偿记录 | 永久 | 涉及工伤保险法规，不允许自动清理 |
| 施工日志 | 3 年 | |
| 工资条 | 5 年 | |
| 工资确认存证（含签署 PDF） | 劳动合同终止后 5 年 | 实际为不定期，到员工离职才开始计算 |
| 电子签名档案 | 永久 | 不允许删除 |
| 审批历史 | 5 年 | |
| 系统操作日志 | 2 年 | |

### 3.2 到期处理流程

```
到期前 30 天：系统通知 CEO（待办角标 + 系统通知）
CEO 操作选项：
  a. 延期（指定延期时长）
  b. 立即备份导出后删除
  c. 忽略（到期自动删除）

到期自动清理：
  删除业务记录 → 查找并删除附件 → 记录审计日志
  附件删除失败 → 进入重试队列 → 多次失败转人工处理
```

## 4. 表单字段配置

### 4.1 请假申请表单（LEAVE）

| 字段 | 类型 | 必填 | 选项 |
|------|------|------|------|
| 请假类型 | 下拉选择 | ✓ | 年假 / 事假 / 病假 / 婚假 / 产假 / 其他 |
| 开始日期 | 日期选择 | ✓ | — |
| 结束日期 | 日期选择 | ✓ | — |
| 请假天数 | 只读（自动计算） | — | — |
| 请假原因 | 文本域 | ✓ | 最多 500 字 |
| 附件 | 文件上传 | 可选 | 最多 3 个，支持图片 / PDF |

### 4.2 加班申请表单（OVERTIME）

| 字段 | 类型 | 必填 | 选项 |
|------|------|------|------|
| 加班日期 | 日期选择 | ✓ | — |
| 加班时长 | 数字输入 | ✓ | 单位：小时，最多 24 |
| 加班类型 | 下拉选择 | ✓ | 平日加班 / 周末加班 / 节假日加班 |
| 加班原因 | 文本域 | ✓ | 最多 500 字 |

### 4.3 工伤补偿表单（INJURY）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 受伤日期 | 日期选择 | ✓ | — |
| 受伤部位 | 文本输入 | ✓ | — |
| 受伤类型 | 下拉选择 | ✓ | 轻伤 / 重伤 / 骨折 / 其他 |
| 事故描述 | 文本域 | ✓ | 最多 1000 字 |
| 医疗单据 | 文件上传 | ✓ | 最多 10 个 |
| 补偿金额 | 数字输入 | ✓ | 单位：元 |

### 4.4 施工日志表单（CONSTRUCTION_LOG）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属项目 | 下拉选择 | ✓ | 仅本人所属项目 |
| 日志日期 | 日期选择 | ✓ | 默认今日 |
| 工作内容 | 文本域 | ✓ | 最多 1000 字 |
| 工作时长 | 数字输入 | ✓ | 单位：小时 |
| 现场照片 | 图片上传 | 可选 | 最多 5 张 |
