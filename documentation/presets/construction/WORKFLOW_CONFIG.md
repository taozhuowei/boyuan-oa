# 博渊·建筑工程版 — 工作流与业务规则配置

> **文档职责**：定义建筑工程版各业务类型的审批流、状态机、薪资计算规则、数据保留策略和表单字段。
>
> **目标读者**：产品经理、后端开发者（种子数据与引擎配置参考）。
>
> **不包含**：员工/岗位/角色体系（见 `ORG_CONFIG.md`）；页面布局（见 `UI_DESIGN.md`）。

---

## 1. 审批流配置

### 1.1 请假申请（LEAVE）

**流程：**
```
[员工提交]
    ↓ PENDING
[节点1: 直系领导 初审，项目范围内；无直系领导时转 CEO]
    ↓ APPROVING
[节点2: CEO 终审，全局]
    ↓
APPROVED ──→ ARCHIVED（归档，自动纳入薪资结算扣款计算）
REJECTED ──→ 申请人可查看驳回原因，并重新提交（新单据，历史保留）
```

**状态机：**
```
DRAFT → PENDING → APPROVING → APPROVED → ARCHIVED
                      ↓
                  REJECTED（可重新发起）
```

**配置参数：**

| 参数 | 值 |
|------|---|
| `businessType` | `LEAVE` |
| 节点数 | 2 |
| 节点1 | 角色: 直系领导（`directSupervisor`）\| 范围: `PROJECT_OR_DEPT` |
| 节点2 | 角色: `ceo` \| 范围: `GLOBAL` |
| 驳回处理 | 单据状态置为 `REJECTED`，原单据完整保留，申请人可提交新单据 |
| 追溯申报 | 任意时刻可补申报已发生但未申请的请假；审批通过后扣款计入**审批完成当月**工资单 |

---

### 1.2 加班申请（OVERTIME）

> **加班采用通知制**：所有员工（OFFICE / LABOR）不可自主申报加班，统一由 PM 或 CEO 发起加班通知。

**正常路径（PM 发起）：**
```
[PM 发起加班通知（选项目/成员/日期/说明）]
    ↓
[通知发送给选中员工，员工默认同意；可在通知中拒绝并说明原因，由 PM 终审拒绝是否接受]
    ↓ PENDING
[CEO 终审]
    ↓ APPROVED → ARCHIVED（写入加班记录，计入薪资结算）
```

**CEO 直接发起：**
```
[CEO 发起加班通知]
    ↓
[通知发送给选中员工，员工默认同意；可拒绝并说明原因，由 PM 终审]
    → 自动 APPROVED（CEO 发起无需额外终审）
    → ARCHIVED
```

**员工自补路径（PM 漏报时）：**
```
[员工发现 PM 未提交加班通知，自行补报加班申请]
[需上传证明材料（如现场照片、聊天记录截图等）]
    ↓ PENDING
[PM 初审（确认该员工确实加班）]
    ↓ APPROVING
[CEO 终审]
    ↓ APPROVED → ARCHIVED
```

> **加班跨月结算**：审批完成后计入**审批完成当月**工资单，与请假追溯规则一致。

**配置参数：**

| 参数 | 值 |
|------|---|
| `businessType` | `OVERTIME` |
| 发起方 | `project_manager` \| `ceo`（不接受员工自发起，仅自补路径除外） |
| 员工自补路径 | 需附证明材料，节点1: PM，节点2: CEO |
| CEO 发起 | 自动归档，无终审节点 |

### 1.3 工伤补偿（INJURY）

**代录规则：** 任何员工均可代他人提交工伤补偿申请，无部门或项目限制，填写被提交人工号即可。代录人身份影响节点1路由（见下），但代录人本身不参与审批流程。

**初审路由规则（节点1动态确定，详见 `ROLE_CONFIG.md §4`）：**
- worker 本人提交 → 节点1: 同项目 project_manager
- project_manager 代录 → **跳过节点1**，直接进入节点2（CEO）
- finance 代录 → 节点1: 同项目 project_manager
- 其他员工/劳工代录 → 节点1: 同项目 project_manager
- 无项目经理时（任意情形）→ 节点1: CEO（兜底）

**流程：**
```
[提交]
    ↓ PENDING
[节点1: project_manager 初审（依路由规则，可被跳过）]
    ↓ APPROVING
[节点2: CEO 终审]
    ↓
APPROVED ──→ ARCHIVED（记录工伤事实，进入待理赔状态）
                ↓（异步，保险理赔完成后）
            财务录入理赔金额，关联至目标薪资周期（可跨月）
REJECTED ──→ 申请人可修改后重新提交
```

> **理赔金额录入**：不在申请表单中填写，由财务在工伤审批归档后、保险理赔结束时录入，指定计入哪个薪资周期。理赔录入与审批归档可以不在同一月份。

**配置参数：**

| 参数 | 值 |
|------|---|
| `businessType` | `INJURY` |
| 节点1 | 动态（见路由规则），范围: `PROJECT` |
| 节点2 | 角色: `ceo` \| 范围: `GLOBAL` |
| 薪资联动 | 财务手动录入理赔金额后写入指定周期 `PayrollSlip.injury` 字段 |
| 节点1跳过条件 | 提交人 role == `project_manager`（skipCondition: `SUBMITTER_ROLE_MATCH`） |

### 1.4 施工日志（CONSTRUCTION_LOG）

> **施工日志是独立系统**，不影响薪资结算，不纳入预结算校验。

#### 1.4.1 申报周期配置

| 参数 | 默认值 | 修改权限 |
|-----|--------|---------|
| `logSubmissionCycle` | 每天（DAILY） | CEO 直接修改（写入 `Project.logReportCycleDays`），无需发起审批 |

周期选项：DAILY / EVERY_2_DAYS / WEEKLY / CUSTOM

#### 1.4.2 工作项模板管理

PM 和 CEO 可在项目级别配置**工作项模板**，供劳工填报时快速选择：

```
WorkItemTemplate:
  projectId        所属项目（模板绑定到项目）
  templateName     模板名称
  items[]:
    itemName       工作项名称（如"焊接口"）
    defaultUnit    默认单位（如"个"）
```

- 可基于现有模板派生出新模板（复制后修改）
- 可完全自定义新模板
- 劳工填日志时优先从模板选择，也可追加非模板自定义项
- 无模板时退回自由文本填写

#### 1.4.3 进度里程碑配置

PM 在项目初始化时定义若干**关键进度节点**：

```
ProjectMilestone:
  projectId
  milestoneOrder       顺序编号（1, 2, 3...）
  milestoneName        节点名称（如"基础浇筑完成"）
  actualCompletionDate 实际完成日期（PM 拖动进度条确认完成时系统自动记录）
  status               PENDING / COMPLETED
```

- 里程碑**只记录实际完成日期**，不设计划日期（工程预测偏差大，计划日期价值低）
- PM 拖动进度条将节点标记为 COMPLETED，系统写入 `actualCompletionDate`，CEO 实时可见
- 完成时间戳用于后续阶段复盘（哪个阶段耗时过长）

#### 1.4.4 单条日志流程

```
[劳工在申报周期内提交日志]
  填写：所属项目 + 日志日期 + workItems（模板选择/自定义）+ 补充说明 + 照片
    ↓ PENDING
[PM 审批：通过 / 驳回（可要求重写）]
PM 可在审批时追加补充内容（独立附注，不覆盖劳工原文）
    ↓ APPROVED / REJECTED

[CEO 追溯驳回（任意时刻）]
ARCHIVED ──→ RECALLED（劳工须重新提交，原记录完整保留）
```

#### 1.4.5 汇总报告流程

```
[PM 完成当期所有日志审批后]
PM 触发"生成汇总报告"
  → 系统聚合指定项目 + 时间段内所有 APPROVED 日志
  → PM 选择可视化组件（每日工作量折线图 / 进度里程碑时间轴 / 工作项汇总表）
  → PM 填写总体说明
  → 提交，系统通知 CEO 查看
  → 自动 ARCHIVED
```

**汇总报告内容：**
- 日志明细列表（按日期 + 劳工排列）
- 工作项汇总表（按工作项名称聚合，标准模板项）
- 可视化：每日工作量折线图（可穿透到该日原始日志）
- 可视化：里程碑时间轴（显示节点完成时间）
- PM 总体说明文字

**CEO 复盘视图：**
- 跨项目、跨时间筛选，系统实时生成聚合图表
- 折线图点击穿透 → 该日该项目所有日志
- 按里程碑区间筛选日志（复盘某阶段）

**状态机：**
```
DRAFT → PENDING → APPROVED → ARCHIVED ←─────┐
                     ↓                       │
                 REJECTED                CEO 追溯驳回
              （可重新发起）              RECALLED
```

**配置参数：**

| 参数 | 值 |
|------|---|
| `businessType` | `CONSTRUCTION_LOG` |
| 节点数 | 1（PM 审批） |
| 节点1 | 角色: `project_manager` \| 范围: `PROJECT` |
| CEO 追溯驳回 | 已归档日志可随时变为 `RECALLED`，写入 operation_log |

### 1.5 薪资结算更正（PAYROLL_CORRECTION）

> 此流程**不是标准单据审批流**，而是一个结算周期解锁流程，由 `PayrollCycle` 状态驱动。

**流程：**
```
[财务发起更正申请，填写更正原因]
    ↓ CORRECTION_PENDING
[CEO 审批]
    ↓
APPROVED:
  周期状态置为 UNLOCKED
  → 财务重新执行结算（生成新版本，版本号 +1）
  → 新版本推送员工重新确认
  → 历史版本完整保留，状态标记为 SUPERSEDED，不可篡改

REJECTED:
  周期保持 LOCKED 状态，更正申请记录驳回原因
  财务可修改原因后再次发起
```

**工资单多版本规则：**
- 每次重新结算生成新的 `PayrollSlip`，版本号递增（v1, v2, v3...）
- 员工只能看到并确认**最新版本**
- 历史版本只读，不接受确认或异议操作
- 所有版本均计入审计日志

---

### 1.6 审批流通用约束

- 当前版本**不支持条件分支**（如金额超过阈值走不同节点）
- **不支持并行会签**
- 审批流变更仅对**变更后新提交**的单据生效，历史在途单据保持原流程配置
- 单据一旦进入 `ARCHIVED`，普通角色不可撤回；**CEO 拥有追溯驳回权限**
- **所有审批节点操作**（提交、审批、驳回、归档、追溯驳回）均写入 operation_log，保留 1 年

## 2. 薪资规则配置

### 2.1 结算周期

以下所有参数均由 sysadmin（初始化时）或 CEO（运营期）在系统配置页调整。

| 参数 | 说明 | 初始默认值 |
|-----|------|-----------|
| 周期类型 | `MONTHLY`（月结） | `MONTHLY` |
| 起始日 | 每周期起始日 | 1日 |
| 结束日 | 每周期结束日 | 月末 |
| 发薪日 | 工资发放日，支持当月或次月 | 当月 25 日 |
| 结算截止日 | 财务须完成结算的日期 | 当月 25 日 |
| 结算提醒 | 结算截止日前 **7 天**，系统向财务推送提醒 | 系统自动 |

> 结算触发由财务**手动发起**，非自动触发；到期提醒仅提醒，不自动执行。
>
> **P3（低优先级）**：日结 / 周结等短周期结算类型，当前仅实现 `MONTHLY`，其他类型保留可配置接口，延后实现。短于一个月的结算周期不设窗口期，结算后直接推送工资条，员工有异议可通过 PAYROLL_CORRECTION 流程上报。

---

### 2.2 预结算流程（窗口期模型）

> **核心原则**：将"阻塞式校验"改为"窗口期锁定"，任何单个员工的未申报不阻塞全员结算。

#### 2.2.1 数据确认窗口期

| 参数 | 默认值 | 修改权限 |
|-----|--------|---------|
| 窗口期时长 | 7 天 | CEO 配置 |
| 触发方式 | 财务手动发起（每个结算周期结束后） | — |

**窗口期内各方操作权限：**

| 角色 | 可操作内容 |
|-----|-----------|
| 员工/劳工 | 查看本人考勤记录；提出考勤异议；提交本人的请假申请（窗口期关闭后不再受理新申请） |
| PM | 发起加班通知；审批日志；补充处理本期未完成的审批 |
| 财务 | 查看考勤数据全览；标记/修正异常考勤 |
| CEO | 查看全局状态；配置窗口期时长（`windowDays`）；窗口期**不可提前关闭**，到期自动锁定 |

#### 2.2.2 窗口期结束后的数据锁定规则

| 数据类型 | 锁定处理 |
|---------|---------|
| 考勤异常（未处理） | 按"缺勤"处理，从当月工资中扣款 |
| 加班（未申报/未审批完成） | 不计入本月；审批跨月完成后计入**审批完成当月** |
| 请假（窗口期内提交但审批未完成） | 审批完成后计入**审批完成当月**（提交时间决定归属，不是审批时间） |
| 请假（追溯补申报，任意时刻） | 审批完成后扣款计入**审批完成当月** |
| 工伤补偿（审批未完成） | 不阻塞本月结算，理赔金额在任意后续周期录入 |

#### 2.2.3 结算前置校验（2项强制）

窗口期锁定后，财务发起结算，`PayrollEngine.settle()` 执行以下 2 项强制校验，**不可豁免**：

| 序号 | 校验项 | 通过条件 |
|-----|--------|---------|
| 1 | 无未处理异议 | 无状态为 `PENDING_REVIEW` 的 `PayrollSlip`（即员工已提交的异议均已处理完毕） |
| 2 | 无进行中计算任务 | 无状态为 `CALCULATING` 的 `PayrollCycle`（即当期无未完成的并发计算任务） |

> **其他状态不阻塞结算**：施工日志未归档、加班审批跨月、工伤理赔未完成，均不阻塞结算。窗口期关闭时按锁定规则（§2.2.2）自动处理，财务无需人工介入。

### 2.3 工资项构成与计算公式

| 工资项 | 计算类型 | 适用对象 | 计算公式 / 规则 |
|-------|---------|---------|----------------|
| 基本工资 | 固定 | 全员 | 取员工等级 `baseSalary`，未配等级则取岗位 `baseSalary` |
| 绩效奖金 | 岗位固定值 | `hasPerformanceBonus=true` 的岗位 | 取岗位/等级 `performanceBase`；财务可设置个人覆盖值（需CEO审批） |
| 加班补贴 | 自动计算 | 全员 | 见下方公式，基数来自岗位 `overtimeBase` 配置 |
| 请假扣款 | 自动计算 | 全员 | 见下方公式，基数来自岗位 `leaveDeductBase` 配置 |
| 工伤补偿 | 手动录入 | LABOR（requiresConstructionLog=true）| 财务在理赔完成后手动录入，关联至指定薪资周期（可跨月） |
| 社保（个人部分） | 自动计算 | 全员 | 见下方"社保模式"说明 |
| 住房公积金（个人） | 自动计算 | 全员 | 基本工资 × 岗位 `housingFundRate`（默认 5%） |
| 个人所得税 | 自动计算 | 全员 | 按**累计预扣法**计算，由平台引擎执行 |
| 实发工资 | 汇总 | 全员 | 基本工资 + 绩效奖金 + 加班补贴 + 工伤补偿 − 请假扣款 − 社保 − 公积金 − 个税 |

**加班补贴公式：**
```
日薪 = overtimeBase（基本工资 或 应发合计，由岗位配置决定）/ 21.75
时薪 = 日薪 / 8

平日加班补贴   = 时薪 × overtimeRateWeekday × 加班小时数
周末加班补贴   = 时薪 × overtimeRateWeekend × 加班小时数
节假日加班补贴 = 时薪 × overtimeRateHoliday × 加班小时数
```

**请假扣款公式：**
```
日薪 = leaveDeductBase / 21.75

扣款 = 日薪 × 请假天数 × LeaveTypeDef.deductRate
```

各假种扣款比例由 `LeaveTypeDef` 表配置（见 §2.3 LeaveTypeDef 默认配置），管理员可按需调整：

| 假种 | 默认 `deductRate` |
|-----|-----------------|
| 年假 | 1.0（全额扣） |
| 事假 | 1.0 |
| 病假 | 0.5 |
| 婚假 | 0.0（不扣薪） |
| 产假 | 0.0 |
| 其他 | 1.0 |

**社保模式（按配置继承链取值，见 `ORG_CONFIG.md §7`）：**

| 模式 | 描述 | 工资条体现 |
|-----|------|----------|
| **COMPANY_PAID（公司代缴）** | 社保由公司代为缴纳，个人部分从实发工资中扣除 | 显示"社保扣款（个人）"，从实发中扣除；可按险种分行展示 |
| **MERGED_SALARY（并入工资）** | 五险一金补贴以合计金额并入工资发放；员工须自行向社保机构办理**灵活就业**参保手续，公司不代缴 | 显示"五险一金补贴（合计）"，不从实发中扣除 |

> **社保配置切换**：切换社保模式（或调整比例）需 CEO 审批，从**下一个完整结算周期**起生效。

**社保比例配置（`SocialInsuranceItem` 表）：**

各险种以独立行存储于 `SocialInsuranceItem` 表，管理员可增删改，引擎遍历所有启用行计算：

| 险种名 | 计算基数 | 默认个人比例 | 模式 |
|-------|---------|------------|------|
| 养老保险 | 基本工资 | 8% | COMPANY_PAID |
| 医疗保险 | 基本工资 | 2% | COMPANY_PAID |
| 失业保险 | 基本工资 | 0.5% | COMPANY_PAID |
| 五险一金补贴 | 基本工资 | 10.5%（合计） | MERGED_SALARY |

> - `COMPANY_PAID` 模式：各险种分项计算，从工资中逐项扣除，工资条按行展示
> - `MERGED_SALARY` 模式：通常只启用一行合计补贴，并入工资发放，工资条显示"五险一金补贴（合计）"
> - 比例可按城市标准修改 `SocialInsuranceItem.employeeRate`，CEO 审批后下一结算周期生效

> **工伤保险、生育保险**：由用人单位（企业）全额缴纳，不从员工工资中扣除，不计入工资单。

### 2.4 薪资档位（SalaryGrade）

sysadmin 初始化时按岗位/等级批量配置薪资档位，员工档案直接关联档位，无需逐人录入基本工资。

> **注意**：薪资档位已整合进岗位（Position）和等级（Level）配置中，不再是独立实体。员工的 baseSalary 取值链：个人覆盖 → 等级 baseSalary → 岗位 baseSalary → 全局默认。

### 2.5 工资确认协议

员工查看工资条并进行电子签名确认时，需同意工资确认协议。

| 参数 | 说明 |
|-----|------|
| 协议内容 | CEO 在系统初始化时可上传（可选），运营期可随时修改 |
| 生效时机 | 修改后的协议从下一个结算周期的工资条确认流程生效 |
| 未上传时 | 使用系统默认协议文本（"本人已阅读并确认以上工资明细，认可以上金额"） |
| 展示位置 | 电子签名确认弹窗中，签名前展示协议全文（含原文链接），员工须**滚动至底部**后签名按钮才可点击 |
| 意图声明 | 员工确认时须看到"我已阅读并同意以上协议内容，本次签名代表本人真实意愿"字样 |

## 3. 数据保留策略

### 3.1 默认保留期

> 所有数据默认保留 **1 年**，到期前 30 天提醒 CEO 备份。延长保留期功能作为后续收费点预留，当前不对外开放。

| 数据类型 | 默认保留期 | 说明 |
|---------|-----------|------|
| 请假记录 | 1 年 | — |
| 加班记录 | 1 年 | — |
| 工伤补偿记录 | 1 年 | 涉及法规合规，建议备份后延期 |
| 施工日志（单条） | 1 年 | — |
| 施工日志汇总报告 | 1 年 | — |
| 工资条 | 1 年 | — |
| 工资确认存证（含 PDF） | 1 年 | 建议在员工离职时手动备份 |
| 电子签名档案 | 1 年 | 涉及法律证明效力，建议备份后延期 |
| 审批历史记录 | 1 年 | — |
| 系统操作日志 | 1 年 | — |

### 3.2 到期处理规则

```
到期前 30 天：
  系统向 CEO 推送通知（待办角标 + 系统通知列表）
  通知内容：数据类型、预计到期日、数量

CEO 操作选项：
  a. 导出后删除 → 触发全量备份导出，导出完成后自动执行清理（推荐）
  b. 忽略       → 到期时自动执行清理，不再提醒

  注：延期功能预留接口，当前 UI 不展示（后续收费功能）

到期自动清理流程：
  定位到期业务记录
    → 查找关联附件元数据（AttachmentMeta）
    → 删除附件物理文件
        ├─ 成功 → 删除数据库记录 → 写入审计日志
        └─ 失败 → 进入重试队列（最多重试 3 次，每次间隔 24h）
                   多次失败 → cleanup_task.status = FAILED
                            → **Sysadmin** 登录 Admin 运营控制台「系统状态」Tab 可见异常提醒
                            → 由 Sysadmin 执行「重试」（排查磁盘/存储问题后）或「标记处理」（人工已处理）
                   **P3**：连续 N 天未处理 FAILED 任务时，通过 SmsService 发送短信提醒至 Sysadmin（接口预留，当前实现为 no-op）
```

## 4. 表单字段定义

### 4.1 请假申请（LEAVE）

**最小计量单位可配置**（系统级配置，初始化时设定）：`小时` / `半天` / `天`

| 字段名 | 类型 | 必填 | 选项 / 约束 |
|-------|------|-----|-----------|
| `leaveType` | 下拉选择 | ✓ | 年假 / 事假 / 病假 / 婚假 / 产假 / 其他 |
| `startDate` | 日期选择 | ✓ | 不早于今日（追溯申报时可选历史日期） |
| `endDate` | 日期选择 | ✓ | 不早于 `startDate` |
| `leaveDays` | 只读 | — | 系统按配置的最小单位自动计算（排除法定节假日和周末） |
| `reason` | 文本域 | ✓ | 最多 500 字 |
| `attachments` | 文件上传 | 可选 | 最多 3 个；支持 jpg / png / pdf；单文件 ≤ 10 MB |
| `isRetroactive` | 只读 | — | 系统标记，追溯申报时为 true |

---

### 4.2 加班申请（OVERTIME）

**PM / CEO 发起通知的字段：**

| 字段名 | 类型 | 必填 | 选项 / 约束 |
|-------|------|-----|-----------|
| `projectId` | 项目选择 | ✓ | 选择关联项目 |
| `overtimeDate` | 日期选择 | ✓ | 不晚于今日 |
| `overtimeType` | 下拉选择 | ✓ | 平日加班 / 周末加班 / 节假日加班 |
| `hours` | 数字输入 | ✓ | 按系统配置的最小单位起步，上限 24 小时 |
| `targetEmployees` | 员工多选 | ✓ | 默认选中项目全部在职成员，可调整 |
| `reason` | 文本域 | ✓ | 最多 500 字 |

**员工自补字段（额外）：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| `evidenceFiles` | 文件上传 | ✓ | 证明材料（照片/截图等），最多 5 个 |

### 4.3 工伤补偿（INJURY）

> 申请表单不包含工伤类型和理赔金额，这两项由财务在保险理赔完成后另行录入。

| 字段名 | 类型 | 必填 | 选项 / 约束 |
|-------|------|-----|-----------|
| `injuryDate` | 日期选择 | ✓ | 不晚于今日 |
| `injuredPart` | 文本输入 | ✓ | 最多 100 字 |
| `description` | 文本域 | ✓ | 最多 1000 字 |
| `medicalRecords` | 文件上传 | ✓ | 最多 10 个；支持 jpg / png / pdf；单文件 ≤ 20 MB |
| `proxyEmployee` | 员工选择 | 代录时必填 | 代录时对所有员工可见；填写被代录人工号 |

### 4.4 施工日志（CONSTRUCTION_LOG）

**劳工填报字段：**

| 字段名 | 类型 | 必填 | 选项 / 约束 |
|-------|------|-----|-----------|
| `projectId` | 下拉选择 | ✓ | 仅列出本人所属的进行中项目 |
| `logDate` | 日期选择 | ✓ | 默认今日；不晚于今日；同一项目同一日期不可重复提交 |
| `workItems` | 动态列表 | ✓ | 至少填写一行；优先从模板选择，也可自定义；每行：目标名称（文字）+ 数量（数字）+ 单位（文字） |
| `workContent` | 文本域 | 可选 | 补充说明，最多 1000 字；与 workItems 合并输出最终日志 |
| `photos` | 图片上传 | 可选 | 最多 5 张；支持 jpg / png；单张 ≤ 10 MB |

> **系统输出规则**：提交后系统将 `workItems` 渲染为表格（目标 / 数量 / 单位），并追加 `workContent` 文字，合并为最终施工日志存档内容。

**PM 审批时额外操作：**
- 驳回：填写驳回原因，要求劳工重新提交
- 追加补充内容：独立附注字段，不覆盖劳工原文；附注随日志一起归档

**汇总报告生成字段（PM 操作）：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| `projectId` | 自动 | — | 当前项目 |
| `dateRangeStart / End` | 自动 | — | 系统默认本期未生成报告的时间段，PM 可调整 |
| `visualComponents` | 多选 | ✓ | 每日工作量折线图 / 里程碑时间轴 / 工作项汇总表（至少选一项） |
| `summary` | 文本域 | ✓ | PM 总体说明，最多 2000 字 |
