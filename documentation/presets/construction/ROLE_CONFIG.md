# 博渊·建筑工程版 — 角色与权限配置

> **文档职责**：定义建筑工程版预置方案的角色体系、权限矩阵和角色分配规则。这是业务配置，不是平台代码。
>
> **目标读者**：产品经理、实施顾问、后端开发者（实现权限引擎时的配置参考）。
>
> **不包含内容**：平台权限引擎的实现原理（见 `platform/ARCHITECTURE.md §5`）；工作流配置（见 `WORKFLOW_CONFIG.md`）。

## 1. 账号体系与初始化流程

```
第零步：Sysadmin  系统初始化（首次部署）
    ↓
    [系统初始化向导]  填写公司信息 → 创建第一个 CEO 账号
                      → 确认默认角色模板 → 确认默认审批流
    ↓
    初始化完成，Sysadmin 退出，CEO 接管

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第一步：CEO / 财务  配置角色权限（角色管理页）
    ↓
    新建角色 → 勾选权限项 → 保存
    （系统内置 5 个角色不可删除，可新增自定义角色）

第二步：CEO / 财务  创建员工档案（员工管理页）
    ↓
    填写员工信息 → 分配角色 → 生成工号 + 初始密码

第三步：员工登录
    ↓
    输入工号 + 密码 → 后端返回员工信息和已分配角色
    → 前端按角色渲染视图，直接进入工作台
    （无角色选择步骤，角色由管理员分配，不可自选）
```

## 2. 角色定义

建筑工程版预置 5 个系统内置角色（不可删除）：

| 角色代码 | 角色名称 | 身份定位 | 员工类型 |
|---------|---------|---------|---------|
| `ceo` | CEO | 企业最高管理者，全局终审，经营数据查看 | OFFICE |
| `finance` | 财务 | 薪资结算、人员管理、财务数据维护 | OFFICE |
| `project_manager` | 项目经理 | 项目范围内初审，施工日志管理 | OFFICE |
| `employee` | 员工 | 普通办公室员工，发起申请 | OFFICE |
| `worker` | 劳工 | 现场施工人员，额外拥有日志和工伤权限 | LABOR |

**扩展规则：**
- CEO 可在角色管理页新增自定义角色，从内置角色权限集中按需勾选
- 自定义角色可被删除，内置角色不可删除、不可修改 `roleCode`
- 当前版本每名员工只分配一个主角色

## 4. 工作台默认视图（按角色）

登录后工作台首页根据角色展示不同内容：

| 角色 | 统计卡片 | 待办入口 | 功能模块 |
|------|---------|---------|---------|
| `ceo` | 员工总数、待审批数、在建项目数、本月支出 | 薪资审批、请假审批 | 全部模块 |
| `finance` | 本月薪资总额、待发放人数、工资异议数 | 薪资结算待处理 | 薪资、员工管理 |
| `project_manager` | 负责项目数、项目成员数、待审批数 | 请假审批、日志审核 | 项目管理、考勤审批 |
| `employee` | 本月请假天数、加班时长、工资条状态 | 确认工资条 | 发起考勤申请、工资条 |
| `worker` | 同 employee | 确认工资条、待提交日志 | 发起考勤申请、施工日志、工资条 |
