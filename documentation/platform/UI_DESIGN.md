## 1. Web 端 Shell 结构

所有业务页面共用同一套 Shell，**系统初始化向导和 Admin 控制台使用独立极简 Layout，不含 LeftNav / TopBar**。

```
+---------------------------------------------------------------+
|  {TopBar}                                                     |
|  "博渊" Logo  |  当前模块名（随路由）  |  待办数 通知数 用户名·角色名 退出 |
+----------+----------------------------------------------------+
|          |                                                    |
| {LeftNav}|  [主内容区]                                        |
|          |  （各业务页面填充此区域）                            |
| 工作台    |                                                    |
| 考勤管理  |                                                    |
| 薪资管理  |                                                    |
| 项目管理  |                                                    |
| 工伤补偿  |                                                    |
| 员工管理  |                                                    |
| 岗位管理  |  [R:ceo / finance]                                 |
| 角色管理  |  [R:ceo]                                           |
| 组织架构  |  [R:ceo]                                           |
| 通知中心  |                                                    |
| 数据管理  |  [R:ceo]                                           |
| 系统配置  |  [R:ceo]                                           |
+----------+----------------------------------------------------+
```

### {TopBar} 规范

| 区域 | 内容 |
|------|------|
| 左侧 | 系统 Logo + "博渊" |
| 中间 | 当前模块名（`$route.meta.title`） |
| 右侧 | 待办数角标（`<Badge>`）/ 通知数角标（`<Badge>`）/ 用户姓名·角色名 / 退出登录 |

### {LeftNav} 规范

- 宽度 220px，可折叠至 64px（图标模式）
- 菜单项按 `position.featureFlags` + `role.permissionCodes` 过滤，不可见项完全隐藏（非置灰）
- 高亮当前路由对应菜单项

## 3. 登录页 (pages/login)

**设计原则**：纯身份验证入口，无角色选择，无演示账号提示（生产环境不暴露账号信息）。使用 uni-app 原生元素（`input` / `view`），不经适配层，保证立即渲染。

### Web 端

```
+----------------------------------------------------------+
|                                                          |
|                      博  渊                              |
|             众维建筑工程 · 企业协同管理系统                |
|                                                          |
|           +--------------------------------+             |
|           |  员工工号                       |            |
|           |  [______________________________]|           |
|           |                                 |           |
|           |  登录密码                        |           |
|           |  [______________________________]|           |
|           |                                 |           |
|           |  [错误提示区域，默认不展示]        |           |
|           |                                 |           |
|           |  [========= 登  录 =========]   |           |
|           +--------------------------------+             |
|                                                          |
+----------------------------------------------------------+
```

**交互：**
- 账号输入框：`type="text"`，`placeholder="请输入工号"`
- 密码输入框：`type="password"`，支持显示/隐藏切换（眼睛图标）
- 错误提示：登录失败时在按钮上方展示"账号或密码错误"，3秒后自动消失
- 登录中：按钮变为 `loading` 状态，禁止重复点击

**登录流程：**
1. `POST /auth/login` → `{ token, user: { role, roleName, displayName, departmentId, positionId, employeeType } }`
2. 写入 `userStore` → 按 role 跳转：
   - `sysadmin` → `/pages/setup/index`（初始化向导或 Admin 控制台）
   - 其他 → `/pages/index/index`

## 4. 系统初始化向导 (pages/setup/wizard) [R:sysadmin]

**设计原则**：全屏独占，无 TopBar / LeftNav。分步骤分页展示，底部固定进度条和导航按钮，像安装应用一样逐步引导，不在同一屏幕展示所有步骤。

### 总体布局

```
+----------------------------------------------------------+
|                   博渊 · 系统初始化                       |
+----------------------------------------------------------+
|                                                          |
|                                                          |
|  [====== 当前步骤内容区（可滚动）======]                  |
|                                                          |
|                                                          |
|                                                          |
+----------------------------------------------------------+
|  ●────○────○────○────○   第 1 步，共 5 步                |
|  公司  账号  角色  审批  保留                              |
|                    [======= 下一步 →  =======]           |
+----------------------------------------------------------+
```

- 顶部：系统标题（不含导航）
- 中部：当前步骤内容，超出屏幕高度时内部滚动，底部进度条固定不滚动
- 底部进度条：圆点（已完成●/当前●/未到○）+ 步骤名 + 上一步/下一步按钮
- 最后一步完成后按钮变为"完成初始化"

## 5. Admin 运营控制台 (pages/setup/dashboard) [R:sysadmin]

**设计原则**：Admin（Sysadmin）不参与任何业务操作，控制台仅展示**系统运行健康状态**和**异常反馈**。业务数据（员工档案、薪资、审批流）由 CEO / 财务管理，Admin 无权访问，企业无需担忧后台侵入业务数据。

```
+----------------------------------------------------------+
|  博渊 · 运营控制台             [退出登录]                 |
+----------------------------------------------------------+
|  <Tabs>  系统状态 | 账号管理 | 系统日志                   |
+----------------------------------------------------------+

[======== 系统状态 Tab ========]

  [服务健康]
  后端服务     ● 正常运行   运行时间: {{uptime}}
  数据库连接   ● 正常       响应时间: {{dbLatency}}ms
  文件存储     ● 正常       已用/总量: {{used}}/{{total}}GB

  [异常提醒]（无异常时不展示）
  <Alert type=error>
  ⚠ 有 {{failedCount}} 个数据清理任务连续失败超过 3 天，需人工处理
  [查看失败任务列表]

  [====== 失败清理任务列表（展开后）======]
  <Table>  数据类型  到期日  失败原因  重试次数  操作

  [备份状态]
  上次自动备份    {{lastBackupTime}} — ● 成功
  <Button>立即备份（手动触发全量导出）

[======== 账号管理 Tab ========]
  说明：仅可重置密码或禁用账号，不可查看业务数据

  <Table>  工号  姓名   角色   状态       最近登录   操作

  约束：不展示员工个人信息（部门、薪资、审批记录）

  说明：只读，展示平台级操作（账号创建/禁用、配置变更），不展示业务审批内容
```
