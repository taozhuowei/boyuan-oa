# 博渊 OA 平台 — 平台级 UI 设计

> **文档职责**：定义博渊平台层的 UI 规范，包括 Web / MP Shell 结构、通用页面（登录、系统管理控制台、待办、我的）的布局，以及组件命名规范。
>
> **目标读者**：前端开发者、UI 设计师。
>
> **不包含内容**：具体业务模块页面（考勤、薪资、项目等）的布局设计——见 `presets/construction/UI_DESIGN.md`。

## 2. Web 端 Shell 结构

每个业务页面共用同一套 Shell：

```
+---------------------------------------------------------------+
|  {TopBar}                                                     |
|  Logo "博渊"  |  当前模块名  |  待办角标 通知角标 用户名·角色 退出|
+----------+----------------------------------------------------+
|          |                                                    |
| {LeftNav}|  [主内容区]                                        |
|          |  （各业务页面的具体布局）                            |
| 工作台   |                                                    |
| 考勤管理 |                                                    |
| 薪资管理 |                                                    |
| 项目管理 |                                                    |
| 员工管理 |                                                    |
| 角色管理 |                                                    |
| [R:ceo/  |                                                    |
|  finance]|                                                    |
| 数据管理 |                                                    |
| [R:ceo]  |                                                    |
+----------+----------------------------------------------------+
```

### {TopBar} 规范

| 区域 | 内容 |
|------|------|
| 左侧 | 系统 Logo + "博渊" |
| 中间 | 当前模块名（随路由变化，如"考勤管理"） |
| 右侧 | 待办数角标 / 系统通知角标 / 用户姓名·角色名 / 退出登录 |

### {LeftNav} 规范

- 宽度固定，可折叠
- 菜单项按角色过滤显示（`{Permission}` 控制）
- 高亮当前页面
- 导航：`uni.navigateTo` 跳转，每个页面包含公共 `{LeftNav}` 组件

## 4. 登录页 (pages/login)

**说明**：纯身份验证入口，无角色选择。直接使用 uni-app 原生元素，不经适配层，保证立即渲染。

```
|         |  [========= 登 录 =========]     |              |
|         |                                  |              |
|         |  演示账号：ceo.demo / 123456      |              |
|         +----------------------------------+              |
```

### MP 端

```
+-------------------------------+
|  博渊                         |
|  众维建筑工程 · 企业协同管理    |
|                               |
|  账号  [__________________]   |
|  密码  [__________________]   |

|                               |
```

**登录流程：**
1. 输入工号 + 密码 → `POST /auth/login`
2. 后端返回 `{ token, user: { role, roleName, ... } }`
3. 写入 `userStore` → 按 role 分发路由：
   - `sysadmin` → `/pages/setup/index`
   - 其他 → `/pages/index/index`

## 8. 组件映射配置规范

新增组件需在 `adapters/config/components.json` 注册，确认 H5 / MP 两端来源：

```json
{
  "组件名": {
    "h5": { "source": "ant-design-vue", "name": "导出名" },
    "mp": { "source": "vant | @/components/cross-platform/组件名", "name": "组件名" }
  }
}
```

**注意：**
- Vant 无 `Row` / `Col`，MP 端需自定义实现
- `Textarea` H5 端 name 应为 `Textarea`，非 `Input.TextArea`
- 适配层仅用于需双端差异化的组件；纯输入页面直接用 uni-app 原生元素
