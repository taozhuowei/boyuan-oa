## 1. Web 端 Shell 结构

所有业务页面共用同一套 Shell，**系统初始化向导和 Admin 控制台使用独立极简 Layout，不含 LeftNav / TopBar**。

```
+---------------------------------------------------------------+
|  {TopBar}                                                     |
|  {CompanyLogo}  |  当前模块名（随路由）  |  🔔通知 · 待办角标 · 用户名·角色名 · 退出 |
+----------+----------------------------------------------------+
|          |                                                    |
| {LeftNav}|  [主内容区]                                        |
|          |  （点击一级菜单后，多页模块先显示功能宫格；          |
| 工作台    |   单页模块直接展示内容）                            |
| 考勤管理  |                                                    |
| 薪资管理  |                                                    |
| 项目管理  |                                                    |
| 工伤补偿  |  [R:labor]                                         |
| 员工管理  |  [R:ceo / finance]                                 |
| 岗位管理  |  [R:ceo / finance]                                 |
| 角色管理  |  [R:ceo]                                           |
| 组织架构  |  [R:ceo]                                           |
| 数据管理  |  [R:ceo]                                           |
| 系统配置  |  [R:ceo]                                           |
+----------+----------------------------------------------------+
```

> **通知入口**：移至 TopBar 右侧 🔔 图标（带未读角标），不在左侧导航列出。

### {TopBar} 规范

| 区域 | 内容 |
|------|------|
| 左侧 | `{CompanyLogo}`：初始化时配置，未上传显示默认图；CEO 可在系统配置中重新上传 |
| 中间 | 当前模块名（`$route.meta.title`） |
| 右侧 | 🔔 通知图标（`<Badge>` 显示未读数）/ 待办数角标（`<Badge>`）/ 用户姓名·角色名 / 退出登录 |

> ⚠ **默认密码提醒**：用户首次登录且未修改初始密码时，TopBar 下方显示全局警告横幅：
> `<Alert type=warning>` "您仍在使用初始密码，存在安全风险，[立即修改]"
> 修改密码后横幅消失；用户可手动关闭（关闭后当日不再显示）。

### {LeftNav} 规范

- 宽度自适应（建议 `220px` 或视口宽度约 `14%`），可折叠至 `64px`（图标模式）
- 菜单项按 `position.featureFlags` + `role.permissionCodes` 过滤，不可见项完全隐藏（非置灰）
- 当前路由对应菜单项加粗显示，图标使用强调色，不强制高亮背景底色（可按设计稿定制）
- 多页模块（如考勤管理）点击后在主内容区展示子功能宫格，左侧菜单保持一级

## 3. 登录页 (pages/login)

**设计原则**：纯身份验证入口，无角色选择，无演示账号提示。使用 uni-app 原生元素（`input` / `view`），不经适配层，保证立即渲染。

**品牌原则**：登录页以企业名称为主视觉，不突出"博渊"平台名称；右下角显示小字"Powered by 博渊"。

### Web 端

```
+----------------------------------------------------------+
|                                                          |
|              {{companyName}}                             |
|              企业协同管理系统                             |
|                                                          |
|           +--------------------------------+             |
|           |                               |             |
|           |  工号 / 手机号                 |             |
|           |  <CustomInput>               |             |
|           |  type="text" placeholder="请输入工号或手机号" |
|           |                               |             |
|           |  登录密码                      |             |
|           |  <CustomInput>               |             |
|           |  type="password" 右侧眼睛图标切换明暗         |
|           |                               |             |
|           |  [错误提示区域，默认隐藏]       |             |
|           |  <ErrorText> 账号或密码错误，3s 后自动消失     |
|           |                               |             |
|           |  [========= 登  录 =========] |             |
|           |  <PrimaryButton> loading状态防重复点击        |
|           |                               |             |
|           |              忘记密码？        |             |
|           |  <TextLink> → 忘记密码流程（§3.1）           |
|           +--------------------------------+             |
|                                                          |
|                              Powered by 博渊            |
+----------------------------------------------------------+
```

**输入框说明：**
- 账号框：`<CustomInput>`，支持工号（字母+数字）和手机号（11位纯数字）两种格式，后端统一识别
- 密码框：`<CustomInput type="password">`，右侧眼睛图标切换显示/隐藏

**交互：**
- 登录中：按钮变为 `loading` 状态，禁止重复点击
- 登录失败：错误文本显示 3 秒后自动消失
- 登录成功：写入 `userStore` → 按 role 跳转

**登录流程：**
1. `POST /auth/login` body: `{ "identifier": "工号或手机号", "password": "..." }`
2. 返回 `{ token, user: { role, roleName, displayName, departmentId, positionId, employeeType } }`
3. 跳转逻辑：
   - `sysadmin` → `/pages/setup/index`（初始化向导或 Admin 控制台）
   - 其他 → `/pages/index/index`

### MP 端

```
+-------------------------------+
|                               |
|  {{companyName}}              |
|  企业协同管理系统              |
|                               |
|  工号 / 手机号                 |
|  <input type="text">          |
|  placeholder="请输入工号或手机号" |
|                               |
|  登录密码                     |
|  <input type="password">      |
|  （含眼睛图标）                |
|                               |
|  [========= 登  录 =========] |
|                               |
|       忘记密码？               |
|   <text> → 忘记密码流程       |
|                               |
|              Powered by 博渊  |
+-------------------------------+
```

**错误提示**：使用 `uni.showToast({ icon: 'none', title: '账号或密码错误' })` 顶部轻提示，不在页面内展示错误文本。

### 3.1 忘记密码流程（弹窗 / 独立页）

通过手机号验证码重置密码，4 步完成：

```
步骤 1：验证手机号
+----------------------------------+
|  忘记密码                         |
|                                   |
|  手机号                           |
|  <CustomInput type="tel">         |
|                                   |
|  [发送验证码]（60s 倒计时防重发）   |
+----------------------------------+

步骤 2：输入验证码
+----------------------------------+
|  输入验证码                        |
|  <OtpInput> 6 位数字输入框         |
|  验证码 10 分钟内有效              |
+----------------------------------+

步骤 3：设置新密码
+----------------------------------+
|  新密码                           |
|  <CustomInput type="password">   |
|  确认新密码                       |
|  <CustomInput type="password">   |
|                                   |
|  密码要求：6–20 位，含字母和数字   |
+----------------------------------+

步骤 4：完成
+----------------------------------+
|  ✓ 密码重置成功                   |
|  [返回登录]                       |
+----------------------------------+
```

**接口：**
- `POST /auth/send-reset-code` body: `{ phone }`（限频：60s 内只能发一次）
- `POST /auth/verify-reset-code` body: `{ phone, code }` → 返回 `resetToken`（5 分钟有效）
- `POST /auth/reset-password` body: `{ resetToken, newPassword }`

## 4. 系统初始化向导 (pages/setup/wizard) [R:sysadmin]

**设计原则**：全屏独占，无 TopBar / LeftNav。每步内容控制在一屏内（不可滚动），通过下一步按钮或左右滑动切换，底部固定进度条和导航按钮，像安装应用一样逐步引导。

### 总体布局

```
+----------------------------------------------------------+
|                   博渊 · 系统初始化                       |
+----------------------------------------------------------+
|                                                          |
|  [====== 当前步骤内容区（固定高度，内容须适配一屏）======] |
|                                                          |
+----------------------------------------------------------+
|  ●────○────○────○────○   第 1 步，共 5 步                |
|  公司  账号  角色  审批  保留                              |
|                    [======= 下一步 →  =======]           |
+----------------------------------------------------------+
```

- 顶部：系统标题（不含导航）
- 中部：当前步骤内容，**不可滚动**，内容量须控制在一屏内
- 底部进度条：圆点（已完成 ●/当前 ●/未到 ○）+ 步骤名 + 上一步/下一步按钮
- 最后一步完成后按钮变为"完成初始化"
- 步骤间支持左右滑动切换（`SwipeAction` 或 `PageContainer`）

## 7. 通知中心 (pages/notifications) [所有角色]

### Web 端


### 9.1 修改手机号流程（弹窗 3 步）

```
步骤 1：身份验证
+----------------------------------+
|  修改手机号 — 验证身份             |
|                                   |
|  当前手机号：{{phone前三位}}****{{phone后四位}} |
|                                   |
|  验证方式：                       |
|  ○ 发送验证码到当前手机号          |
|  ○ 输入当前登录密码               |
|  （二选一）                       |
|                                   |
|  [验证码输入框 / 密码输入框]       |
|                                   |
|  [下一步]                         |
+----------------------------------+

步骤 2：填写新手机号
+----------------------------------+
|  修改手机号 — 新手机号             |
|                                   |
|  新手机号 *                       |
|  <Input type="tel">               |
|                                   |
|  验证码 *                         |
|  <Input>  [发送验证码]（60s倒计时）|
|                                   |
|  [下一步]                         |
+----------------------------------+

步骤 3：完成
+----------------------------------+
|  ✓ 手机号已更新                   |
|                                   |
|  新手机号：{{newPhone前三位}}****{{newPhone后四位}} |
|                                   |
|  [完成]                           |
+----------------------------------+
```

**接口：**
- `POST /employees/me/phone/send-verify-code` — 发送当前手机号验证码
- `POST /employees/me/phone/verify-identity` body: `{ code? / password? }` → 返回 `changeToken`
- `POST /employees/me/phone/send-new-code` body: `{ changeToken, newPhone }`
- `PUT /employees/me/phone` body: `{ changeToken, newPhone, code }`
