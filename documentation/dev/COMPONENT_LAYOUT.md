# 组件布局设计文档

## 图例

- `[========]` - 页面容器/卡片
- `<Component>` - 组件库组件
- `{Custom}` - 自定义组件
- `|  文本  |` - 静态内容
- `{{data}}` - 动态数据绑定
- `[R:角色]` - 角色可见标记

---

## 账号建立流程

```
第一步：CEO/财务 配置角色
    ↓
    [角色管理页] 创建角色（如：项目经理、财务专员）
    ↓ 分配权限（审批权限、查看权限、操作权限）

第二步：CEO/财务 配置人员
    ↓
    [员工管理页] 添加员工信息 + 分配角色
    ↓ 生成登录账号（工号/初始密码）

第三步：员工登录使用
    ↓
    [登录页] 输入工号密码
    ↓ 系统根据分配的角色渲染对应视图
```

**关键理解：**
- 角色是**配置出来**的，不是选择出来的
- 每个员工**被分配**一个角色，登录后自动获得对应权限
- 角色管理页（pages/role）是**管理后台**，只有CEO/财务可见
- 登录页只验证身份，不选择角色

## 动态数据组件清单

| 组件名 | 用途 | 数据来源 |
|--------|------|----------|
| {UserInfo} | 显示当前用户姓名、职位、部门 | userStore.userInfo |
| {UserAvatar} | 显示用户头像 | userStore.userInfo |
| {RoleBadge} | 显示角色标识 | userStore.userInfo.role |
| {Permission} | 权限控制容器 | userStore.userInfo.role |
| {PendingCount} | 待办事项数量 | API: /todos/count |
| {NoticeList} | 系统通知列表 | API: /notices |
| {StatValue} | 统计数值 | API: /stats/* |
| {ApprovalList} | 待审批列表 | API: /approvals/pending |
| {MyRecordList} | 我的记录列表 | API: /records/my |
| {ProjectList} | 项目列表 | API: /projects |
| {EmployeeList} | 员工列表 | API: /employees |
| {RoleList} | 角色列表 | API: /roles |
| {TimelineData} | 时间轴数据 | API: /timeline/* |

**权限差异：**
- **CEO**：可新增角色、可删除自定义角色、可配置"终审审批"权限
- **财务**：可编辑角色权限、不能删除系统内置角色、不能配置"终审审批"权限

**预置角色（系统内置，不可删除）：**
- 员工：发起请假/加班、查看并确认本人工资条
- 劳工：在员工基础上，可提交施工日志、发起工伤补偿
- 财务：维护人员与薪资配置、执行结算
- 项目经理：处理项目范围内初审、查看项目总览
- CEO：管理全局配置、终审审批、配置角色权限

**自定义角色：**
- CEO可创建新角色，如"财务主管"、"项目总监"等
- 自定义角色可勾选上述权限的任意组合

## 7. 登录页 (pages/login)

**说明：** 员工使用**工号+密码**登录，系统根据**被分配的角色**自动跳转对应视图，无角色选择过程。

```
+----------------------------------------------------------+
|                                                            |
|                     工程智慧办公 OA                        |
|                                                            |
|              +------------------------------+              |
|              |      [========]              |              |

|              |       用户登录               |              |
|              |                              |              |
|              |   工号    <Input>            |              |
|              |                              |              |
|              |   密码    <Input.Password>   |              |
|              |                              |              |
|              |   <Button> 登录              |              |
|              |                              |              |
|              +------------------------------+              |
|                                                            |
+----------------------------------------------------------+
```

**登录流程：**
1. 输入工号密码
2. 后端验证身份，返回用户信息和**被分配的角色**
3. 前端根据角色渲染对应工作台视图
4. 无角色选择步骤
