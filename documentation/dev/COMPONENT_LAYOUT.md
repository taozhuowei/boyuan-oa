# 组件布局设计文档

> **阅读说明：** 本文档定义各页面在 Web 端和 MP 端（企业微信小程序）的完整布局方案。
> 代码实现必须以本文档为准；如有冲突，优先修改代码以对齐文档。
>
> 相关文档：技术架构见 `ARCHITECTURE.md`，开发进度见 `TODO.md`。

## 图例

- `[========]` — 容器 / 卡片区域
- `<Component>` — 组件库组件（AntD / Vant）
- `{Custom}` — 自定义业务组件
- `{{data}}` — 动态数据绑定
- `[R:角色]` — 角色可见标记（ceo / finance / pm / employee / worker）

## 核心业务流程

### 账号体系

```
第零步：Sysadmin  系统初始化（首次部署）
    ↓
    [系统初始化向导]  填写公司信息 → 创建第一个 CEO 账号
                      → 确认默认角色模板 → 确认默认审批流
    ↓
    初始化完成后，Sysadmin 退出，交由 CEO 接管业务配置
    （Sysadmin 不参与任何业务流程，仅在维护时使用）

────────────────────────────────────────────────

第一步：CEO / 财务  配置角色
    ↓
    [角色管理页]  新建角色 → 勾选权限项 → 保存
                  （系统内置5个角色不可删除，可扩展自定义角色）

第二步：CEO / 财务  创建人员档案
    ↓
    [员工管理页]  填写员工信息 → 分配角色 → 生成工号 + 初始密码

第三步：员工登录
    ↓
    [登录页]  输入工号 + 密码
    ↓
    后端验证身份，返回员工信息和被分配的角色
    ↓
    前端按角色渲染对应视图，直接进入工作台
    （无角色选择步骤）
```

**关键约束：**
- `sysadmin` 登录后**跳转系统管理控制台**（`/pages/setup`），不进入工作台
- 角色由管理员**配置并分配**，员工**不能自选角色**
- 同一员工只有一个角色（当前版本，未来可扩展多角色）
- 系统根据 `userStore.userInfo.role` 自动判断权限和视图

## 平台布局基准

### Web 端（PC / 平板 / 手机浏览器）

每个页面共用同一套 Shell 结构：

```
+---------------------------------------------------------------+
|  [顶部状态栏]                                                  |
|  Logo + 系统名  |  当前模块名  |  待办角标 通知角标 用户 退出   |
+----------+----------------------------------------------------+
|          |                                                    |
|  [左侧   |  [主内容区]                                        |
|  导航栏] |                                                    |
|          |  （各页面具体布局见下文）                           |
| 工作台   |                                                    |
| 考勤管理 |                                                    |
| 薪资管理 |                                                    |
| 项目管理 |                                                    |
| 员工管理 |                                                    |
| 角色管理 |                                                    |
| [R:ceo/  |                                                    |
|  finance]|                                                    |
| 数据管理 |                                                    |
| [R:ceo]  |                                                    |
+----------+----------------------------------------------------+
```

**顶部状态栏（所有页面共用）：**
| 区域 | 内容 |
|------|------|
| 左侧 | 系统 Logo + "博渊" |
| 中间 | 当前模块名（如"考勤管理"），随路由变化 |
| 右侧 | 待办数角标 / 系统通知角标 / 用户姓名 · 角色名 / 退出登录 |

**左侧导航栏（所有页面共用）：**
- 宽度固定，可折叠
- 菜单项按角色过滤显示（`{Permission}` 组件控制）
- 高亮当前页面
- 导航实现：`uni.navigateTo` 跳转对应 page，每个 page 包含公共 `{LeftNav}` 组件

---

### MP 端（企业微信小程序）

每个页面共用同一套 Shell 结构：

```
+-------------------------------+
|  [顶部信息栏]                  |

|                               |
|  [页面内容区]                  |

|  [底部 Tab 栏]                |
|  首页  |  待办  |  我的        |
```

**底部 Tab 栏（固定3项）：**
| Tab | 路由 | 所有角色可见 | 说明 |

| 我的 | `pages/mine/index` | ✓ | 个人信息、工资条、设置 |

**首页宫格功能区按角色显示的模块入口：**
| 模块 | employee | worker | finance | pm | ceo |
|------|----------|--------|---------|-----|-----|

| 施工日志 | ✗ | ✓ | ✗ | ✗ | ✗ |
| 工伤补偿 | ✗ | ✓ | 代录入 | 代录入 | ✓ |
| 项目管理 | ✗ | ✗ | ✗ | ✓ | ✓ |
| 员工管理 | ✗ | ✗ | ✓ | ✗ | ✓ |

## 1. 登录页 (pages/login)

**说明：** 纯身份验证入口，无角色选择。后端返回员工信息和角色，前端直接进入工作台。

### Web 端

```
+----------------------------------------------------------+
|                                                          |
|                      博 渊                               |
|             众维建筑工程 · 企业协同管理系统                |
|                                                          |
|         +----------------------------------+              |
|         |  账号登录                         |              |
|         |                                  |              |
|         |  账号   [___________________]    |              |
|         |                                  |              |
|         |  密码   [___________________]    |              |
|         |                                  |              |
|         |  [========= 登 录 =========]     |              |
|         |                                  |              |
|         |  演示账号：ceo.demo / 123456      |              |
|         +----------------------------------+              |
|                                                          |
+----------------------------------------------------------+
```

**登录流程：**
1. 输入工号 + 密码 → `POST /auth/login`
2. 后端返回 `{ token, user: { role, roleName, ... } }`
3. 写入 `userStore` → `uni.redirectTo('/pages/index/index')`
4. 工作台按 `role` 渲染对应视图

## 组件映射配置

新增组件需在 `adapters/config/components.json` 注册，并确认 H5 / MP 两端的来源：

```json
{
  "组件名": {
    "h5": { "source": "ant-design-vue", "name": "导出名" },
    "mp": { "source": "@/components/cross-platform/组件名", "name": "组件名" }
  }
}
```

**注意事项：**
- `Row` / `Col` 的 MP 端来源需替换为自定义实现（Vant 无此组件）
- `Textarea` H5 端来源应为 ant-design-vue 的 `Input.TextArea`，当前 components.json 有误，待修正
- 适配层仅用于需要双端差异化的组件；登录、我的等纯输入页面直接使用 uni-app 原生元素
